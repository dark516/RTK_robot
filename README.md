## О проекте

**RTK_robot** — это набор ROS 2 workspace’ов, скриптов и вспомогательных файлов, которые использовались для решения заданий хакатонов из цикла **«Кубок РТК: Высшая лига»**.

В репозитории представлены решения для трёх направлений:

1. **Движение по городу**
   Автономная навигация по городскому полигону с дорожной разметкой, зданиями и знаками.

2. **Теплица — сбор урожая**
   Автономный сбор плодов в модели теплицы. В репозитории направление называется `Frob_ogorod`.

3. **Месторождение нефти**
   Обследование макета нефтяного месторождения: движение по линии, поиск протечек, распознавание манометров, ArUco/QR-меток и других объектов.

Решения создавались как прикладной слой над образовательной платформой Frob: базовая ROS 2-инфраструктура, драйверы и механика робота берутся из основного проекта, а здесь хранится код, адаптированный под конкретные соревновательные полигоны.

## О соревнованиях Кубок РТК: Высшая лига

**Кубок РТК: Высшая лига** — это соревнования автономных роботов в формате тематических хакатонов. Участникам необходимо подготовить робота, который самостоятельно выполняет задания на специальном полигоне.

<img width="1280" height="853" alt="photo_2026-04-07_12-24-43" src="https://github.com/user-attachments/assets/6cfe035b-cea9-4ec3-997c-55f714a55dc1" />


Особенность этих соревнований в том, что задача обычно совмещает сразу несколько направлений робототехники:

* автономное движение;
* навигация по линии, карте, разметке или стенам;
* техническое зрение;
* распознавание объектов;
* обработка дорожных знаков, маркеров, QR-кодов или ArUco;
* работа с манипулятором;
* принятие решений в зависимости от условий полигона;
* выполнение задания за ограниченное время.

Для участия требуется заранее собранный автономный робот с питанием на борту. В рамках Высшей лиги робот должен быть достаточно компактным и лёгким, но при этом способным нести вычислительный модуль, камеру, датчики и дополнительные механизмы.

## Робот Frob

Для выполнения заданий использовался модульный образовательный робот **Frob**.

<img width="491" height="508" alt="433066577-4e6b15c5-6662-47f5-8419-bada88828910" src="https://github.com/user-attachments/assets/7fe311a1-2656-4d1a-9311-4a250e891358" />


Frob — это мобильная робототехническая платформа, построенная вокруг ROS 2. Она подходит для обучения, экспериментов и участия в соревнованиях, потому что позволяет быстро менять конфигурацию под конкретную задачу.

### Аппаратная часть

В конфигурации для хакатонов использовались:

* **Raspberry Pi 5** как основной вычислительный модуль;
* **Ubuntu 24.04** как операционная система на Raspberry Pi;
* **ROS 2** как основная система взаимодействия между модулями;
* **Arduino** как низкоуровневый контроллер;
* **Motor Shield** для управления моторами;
* моторы с энкодерами;
* камера для задач компьютерного зрения;
* дополнительные датчики в зависимости от задания;
* сменные механические модули и обвесы;
* манипулятор для задач сбора урожая;
* элементы крепления для камеры, датчиков и вспомогательных модулей.

### Почему Frob удобен для хакатонов

Главная особенность платформы — модульность. Робота можно быстро адаптировать под разные полигоны:

* для города — поставить камеру так, чтобы видеть разметку и дорожные знаки;
* для теплицы — добавить манипулятор и настроить камеру под плоды;
* для нефтяного полигона — настроить распознавание манометров, протечек, ArUco и QR-меток;
* для навигационных задач — использовать линию, карту, одометрию, лидар или комбинацию датчиков.

Благодаря ROS 2 все части робота можно разнести по отдельным узлам: управление моторами, камера, компьютерное зрение, логика движения, интерфейсы и отладочные инструменты работают независимо и обмениваются данными через топики.

## Решаемые задания

### Движение по городу

Папка:

```text
Frob_gorod/
```

Задание **«Движение по городу»** моделирует автономное вождение в городской среде. Робот должен перемещаться по полигону с дорогами, дорожной разметкой, зданиями и дорожными знаками.

Основная идея задания:

* робот стартует из заданной зоны;
* движется по городскому кварталу;
* учитывает дорожные знаки и разметку;
* останавливается у зон посадки, обозначенных знаком «Остановка»;
* выдерживает остановку для условной посадки пассажиров;
* доезжает до зоны высадки;
* останавливается перед знаком «Парковка».

Полигон представляет собой модель городского квартала. В нём есть:

* дорожная разметка;
* Т-образные перекрёстки;
* повороты;
* модели зданий;
* дорожные знаки;
* стартовая зона;
* зоны остановки и парковки.

В рамках решения для Frob важными частями являются:

* построение и использование карты полигона;
* работа с сохранённым маршрутом;
* визуализация дорожных знаков;
* автономная навигация;
* обработка данных от датчиков;
* управление движением через ROS 2;
* использование пакетов одометрии, навигации и фильтрации лидара.

В папке `Frob_gorod` дополнительно находятся:

* `map.pgm` — карта полигона;
* `saved_environment.json` — сохранённое описание окружения;
* `saved_path.json` — сохранённый маршрут;
* `GUI_path.py` — интерфейс для работы с маршрутом;
* `signs_visualizer.py` — визуализация дорожных знаков.

### Теплица / ogorod

Папка:

```text
Frob_ogorod/
```

Задание **«Теплица — сбор урожая»** в репозитории называется `ogorod`.

Роботу необходимо автономно проехать по модели теплицы и собрать указанные плоды. В зависимости от условий конкретного этапа часть плодов нужно сбить в лотки, а один «особый» плод нужно захватить и доставить в специальный ящик.

Особенности задания:

* полигон имитирует теплицу;
* плоды расположены вдоль стен;
* плоды отличаются цветом и формой;
* часть плодов нужно собирать;
* часть плодов нужно игнорировать;
* особый плод нужно захватить манипулятором;
* после выполнения задания робот должен остановиться на финишной ячейке.

В решении использовались два подхода:

1. **Классическое компьютерное зрение на OpenCV**

   * фильтрация по цветам;
   * поиск контуров;
   * определение положения плода в кадре;
   * принятие решения о сборе.

2. **Нейросетевое распознавание на YOLOv8**

   * обучение модели под конкретные плоды;
   * детекция объектов на изображении;
   * проверка возможности запуска на Raspberry Pi 5.

После сравнения подходов для данного задания было принято решение использовать вариант на базе **OpenCV**, так как он оказался более подходящим для условий полигона и доступных вычислительных ресурсов.

Основные части решения:

* получение изображения с камеры;
* распознавание нужных плодов;
* определение момента для действия манипулятора;
* управление движением вдоль теплицы;
* передача результатов обработки изображения в ROS 2;
* запуск основного поведения через `ogorod_main_pkg`.

### Месторождение нефти

Папка:

```text
Frob_neft/
```

Задание **«Месторождение нефти»** моделирует инспекцию промышленного объекта. Робот должен автономно пройти по полигону, обследовать макеты устьевой арматуры, найти признаки протечек и считать данные с манометров.

На полигоне есть:

* линия для навигации;
* ответвления к нефтяным вышкам;
* макеты устьевой арматуры;
* манометры;
* зоны возможных протечек;
* ArUco-маркеры;
* QR-коды;
* гараж;
* карьер;
* декоративные объекты.

Основные задачи робота:

* двигаться по линии;
* корректно проходить повороты и ответвления;
* подъезжать к объектам обследования;
* распознавать манометры;
* определять, находится ли стрелка манометра в красной или зелёной зоне;
* находить места протечек;
* определять номера объектов по ArUco-маркерам;
* не заезжать в запрещённые ответвления;
* завершать маршрут в финишной зоне.

Исторически в этом направлении было несколько версий решения:

* раннее решение на OpenCV для поиска протечек и анализа изображения;
* более позднее решение с использованием кастомно обученных моделей YOLOv8;
* оптимизация запуска на Raspberry Pi 5;
* передача результата распознавания в ROS 2 для дальнейшей логики движения.

В папке `Frob_neft` также сохранена старая версия решения:

```text
first_neft (old)/
```

Это полезно для сравнения подходов и отслеживания эволюции проекта.

## Как работает код

Общая архитектура решений похожа для всех направлений.

```text
Камера / датчики / энкодеры
          │
          ▼
ROS 2 nodes: драйверы, камера, Arduino bridge
          │
          ▼
Компьютерное зрение / распознавание объектов
          │
          ▼
Логика задания: состояние робота, выбор действия
          │
          ▼
Команды движения / команды манипулятору
          │
          ▼
Arduino + Motor Shield + исполнительные механизмы
```

### Основные уровни

#### 1. Низкоуровневое управление

За управление моторами отвечает Arduino. Raspberry Pi отправляет команды через ROS 2-узел `ros2_arduino_bridge`.

Типичный запуск использует последовательный порт:

```text
/dev/ttyUSB0
```

Если Arduino определяется на другом порту, нужно изменить аргумент в launch-файле.

#### 2. Камера

Камера используется для:

* поиска дорожных знаков;
* распознавания плодов;
* поиска протечек;
* считывания манометров;
* распознавания ArUco/QR-маркеров;
* отладки и сохранения изображений.

В репозитории есть пакеты:

```text
pi_camera_ros
image_publisher_pkg
ros2_image_saver
camera_code
```

#### 3. Компьютерное зрение

В зависимости от задания используются:

* OpenCV;
* цветовая сегментация;
* поиск контуров;
* распознавание форм;
* ArUco;
* QR;
* YOLOv8;
* простая логика классификации результата.

Результаты обработки изображения передаются в ROS 2, после чего основной алгоритм робота принимает решение: ехать дальше, повернуть, остановиться, активировать манипулятор или сохранить результат.

#### 4. Основная логика

Основная логика каждого задания находится в соответствующем пакете:

```text
ogorod_main_pkg
neft_main_pkg
frob_bringup / frob_navigation / frob_control
```

Launch-файлы поднимают необходимые узлы и переводят робота в нужное начальное состояние.

## Установка и запуск

> Команды ниже рассчитаны на запуск на роботе Frob или в совместимом ROS 2-окружении.
> Перед запуском на реальном роботе рекомендуется сначала проверить движение с поднятыми колёсами и убедиться, что порт Arduino, камера и ROS_DOMAIN_ID настроены правильно.

### 1. Клонирование репозитория

```bash
git clone https://github.com/dark516/RTK_robot.git
cd RTK_robot
```

### 2. Подключение к роботу

Подключение к Frob обычно выполняется по SSH:

```bash
ssh robot@<robot_ip>
```

Или через доменное имя, если оно настроено:

```bash
ssh robot@r1.frob-robot.online
```

Где `r1` нужно заменить на id конкретной платформы.

### 3. Установка базового окружения

Для работы требуется:

* Ubuntu 24.04;
* ROS 2 Jazzy;
* Python 3;
* colcon;
* OpenCV;
* пакеты ROS 2, используемые Frob;
* доступ к камере;
* доступ к Arduino;
* при использовании нейросетевого решения — зависимости для YOLOv8.

Пример базовых команд:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-colcon-common-extensions
```

После установки ROS 2:

```bash
source /opt/ros/jazzy/setup.bash
```

Для удобства можно добавить строку в `~/.bashrc`:

```bash
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## Запуск направления «Движение по городу»

Папка решения:

```bash
cd RTK_robot/Frob_gorod
```

Сборка:

```bash
colcon build --symlink-install
source install/setup.bash
```

Базовый запуск робота:

```bash
ros2 launch frob_bringup bringup.launch.py
```

Дополнительные инструменты:

```bash
python3 GUI_path.py
python3 signs_visualizer.py
```

Файлы, связанные с маршрутом и картой:

```text
map.pgm
```

## Запуск направления «Месторождение нефти»

Папка решения:

```bash
cd RTK_robot/Frob_neft/ros2_ws
```

Сборка:

```bash
colcon build --symlink-install
source install/setup.bash
```

Запуск:

```bash
ros2 launch neft_main_pkg bringup.launch.py
```

В текущем launch-файле `neft_main_pkg` запускается узел связи с Arduino:

```text
ros2_arduino_bridge / arduino_bridge
```

Также публикуется начальное состояние робота в топик:

```text
/robot_state
```

Для проверки состояния:

```bash
ros2 topic echo /robot_state
```

Перед соревновательной попыткой стоит проверить:

* видимость линии камерой или датчиками;
* качество распознавания манометров;
* работу ArUco/QR-распознавания;
* корректность определения протечек;
* реакцию робота на развилки;
* отсутствие ложного заезда в гараж или карьер;
* стабильность питания Raspberry Pi и Arduino.

## Отладка

### Проверка ROS 2-топиков

```bash
ros2 topic list
```

### Проверка узлов

```bash
ros2 node list
```

### Проверка Arduino

Если робот не двигается:

1. Проверьте, появился ли порт:

```bash
ls /dev/ttyUSB*
ls /dev/ttyACM*
```

2. Проверьте, что launch-файл использует правильный порт.

3. Проверьте права доступа к порту:

```bash
sudo usermod -a -G dialout $USER
```

После этого нужно перелогиниться или перезагрузить робота.

### Проверка камеры

```bash
ros2 topic list | grep image
```

Также можно временно запускать отдельные скрипты компьютерного зрения, если они не требуют полного bringup робота.

### Проверка сети ROS 2

Если компьютер и робот не видят топики друг друга, проверьте `ROS_DOMAIN_ID`:

```bash
echo $ROS_DOMAIN_ID
```

На роботе и на компьютере значение должно совпадать. Например:

```bash
export ROS_DOMAIN_ID=78
```

### Проверка питания

Если Raspberry Pi выключается или робот ведёт себя нестабильно:

* проверьте заряд аккумулятора;
* проверьте питание Raspberry Pi;
* отключите лишние потребители;
* проверьте нагрев;
* проверьте соединения Arduino, Motor Shield и моторов.
